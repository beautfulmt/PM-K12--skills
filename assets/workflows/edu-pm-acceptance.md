# 教育PM功能验收清单工作流

本工作流旨在帮助产品经理在研发提测后或上线前，快速生成结构化的自测清单，确保核心业务流程走通、异常分支被处理、UI高保真还原。

## 适用场景
- 研发提测后的 PM 走查（Sanity Check）
- 上线前的产品功能验收
- 视觉/交互还原度核对

## 执行步骤

### Step 1: 读取 PRD 源文件
1. 询问用户需要验收的 PRD 文件路径（通常位于 `需求文档/` 目录下）。
2. 使用 `view_file` 工具完整读取该 HTML 文件。
3. 提取 PRD 中的：功能模块拆解、业务规则、异常与边界处理表、数据埋点表。

### Step 2: 拆解验收维度
将提取的内容转化为可执行的 Check-list，分类如下：
1. **核心流程**：主线路径是否畅通（如：进入入口 -> 拍照 -> 识别成功 -> 扣减次数 -> 播放语音）。优先级 P0。
2. **边界与异常**：网络断开、超时、无权限、次数用尽、不合规输入等边缘场景的处理。优先级 P0/P1。
3. **UI 还原度**：关键页面的布局、颜色、交互动效、加载状态是否与需求/原型一致。优先级 P1/P2。
4. **数据埋点**：核心转化漏斗的埋点是否触发（如果有条件验收）。

### Step 3: 生成《功能验收清单》HTML
1. 在项目目录下的 `验收清单` 文件夹中创建 HTML。
2. 文件命名规范：`{需求名称}-验收清单.html`。
3. **UI 与交互规范**：
   - 使用现代化、清爽的界面风格（浅色为主，类似 Notion/Linear）。
   - **三态选择器**：每个验收项通过点击在“✅ 通过”、“❌ 未通过”、“⏭️ 跳过”之间切换。
   - **进度统计**：顶部悬浮展示通过率进度条，以及各状态的数量统计。
   - **过滤功能**：支持按状态筛选列表（全部、待验收、通过、未通过）。
   - **模块折叠**：支持按一级模块折叠/展开验收项。
   - **本地存储**：使用 `localStorage` 自动保存验收进度。
   - **导出能力**：底部提供“一键导出验收报告”按钮，生成 txt 或简易汇总文本下载。

## 输出交付物
- 可交互的功能验收清单 (`验收清单/{需求名称}-验收清单.html`)
