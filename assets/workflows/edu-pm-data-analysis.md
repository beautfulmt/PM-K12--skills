# 教育PM数据分析报告工作流

本工作流协助产品经理在功能上线后，快速将原始业务数据转化为结构化、可视化的分析报告，进行漏斗诊断、异常归因，并输出下一步迭代建议。

## 适用场景
- 新功能上线首周/首月的数据复盘
- A/B 测试结果分析
- 日常核心指标监控分析

## 执行步骤

### Step 1: 收集原始数据
1. 接收用户提供的数据源。数据形式可以是：
   - 粘贴的 CSV/Excel 文本数据
   - 业务 BI 看板的截图（需 AI 视觉解析）
   - 用户口述的几个核心关键指标
2. 确认分析的周期（日/周/月）和核心关注点。

### Step 2: 数据结构化与诊断分析
1. **指标概览**：计算核心 KPI（渗透率、转化率、留存率、日均使用次数等），并与预期目标对比，标注升降。
2. **转化漏斗**：梳理用户核心路径流转率，定位断点。
3. **异常归因**：针对偏低或暴跌的指标，结合业务逻辑推导可能的原因（例如：功能隐藏太深、定价门槛过高、UI引导弱）。
4. **专项分析**（可选）：针对不同人群（年级/学科/机型）的交叉分析。

### Step 3: 输出优化建议 (Action Items)
- 基于数据结论，输出 3-5 条具体可落地的产品优化建议，并标记优先级（P0/P1）。

### Step 4: 生成《数据分析报告》HTML
1. 在项目目录下的 `数据分析` 文件夹中创建 HTML 报告。
2. 文件命名规范：`{需求名称}-数据分析.html`。
3. **UI 规范**：
   - 采用专业数据 BI 工具的暗黑主题（深蓝/黑背景，高亮重点色）。
   - **核心卡片**：顶部展示 KPI 概览，包含具体数值、涨跌幅 badge。
   - **可视化图表**：使用原生 SVG 或轻量级图表库（如 ECharts 概念模拟的纯 CSS/SVG）展示趋势折线图、柱状图。
   - **漏斗图**：CSS 绘制的转化漏斗进度条。
   - **洞察模块**：用高亮卡片突出“异常流失”或“超预期”的发现。
   - **行动项**：底部清晰列出下一步迭代计划。

## 输出交付物
- 沉浸式数据分析报告 (`数据分析/{需求名称}-数据分析.html`)
