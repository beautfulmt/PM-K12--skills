v1.7
【问题】：skill 和 workflow 存在多份副本，且部分 PRD 规则带有旧后台口径、错误保存对象、表格 resize 残影风险，容易导致不同 agent 读到不同规范

【思路】：
1. 将 `.agents/workflows/edu-pm-prd.md` 作为项目运行时权威规范，skill assets 作为未来安装模板，根目录 `edu-pm-prd.md` 只作为兼容副本
2. 修正 PRD 保存按钮规则：保存当前 PRD HTML 文件，不写成覆盖原型
3. 去掉默认 `th/td resize: both` 规则，避免表格残影和截图问题
4. 迭代清单从固定 C端/后台 改为“按当前需求实际存在的端组织”，删除 `adminPages` 硬编码
5. `init.sh` 对 workflow/helper 文件采用“更新并备份”策略，避免升级 skill 后旧项目继续使用旧规范


v1.6
【问题】：新增需求的时候经常只更新"直接提到的那一处"（如只改详细方案），忘了同步"需求概述功能清单表""埋点""异常处理""决策对齐表"等其他章节，导致 PRD 前后不一致

【思路】：
1. 在 edu-pm-prd.md 的"5.3 迭代更新规则"里写死一张"变更后必须遍历的 15 个位置清单"
2. 清单覆盖：版本记录 / 功能清单 / 流程图 / 各实际存在端的详细方案 / 异常处理 / 埋点 / 上线计划 / 决策对齐表 / 原型 pages 数组 / 双栏预览下拉项 / 页面映射 等
3. 规则本体强调"反向规则"：原型/流程图的独立变更，也要回查 PRD 是否需要跟改
4. 每轮迭代交付时必须在回复里明确列出"本次同步更新了哪些位置"
5. 更新到工作流 5.3 节


v1.5
【问题】：原型交互体验不够好，多个关联页面（如报告的三个Tab）跳转后Tab栏消失、PRD与原型联动不够丝滑

【思路】：
1. 建立 Tab 切换模式规范：多个平级子页面合并为单页+Tab面板，点击Tab只切内容不跳页
2. 建立 PostMessage 双向通信协议：PRD页面选择器 ↔ 原型 iframe 实时同步
3. 支持 tabAliases 映射：旧页面名自动重定向到对应Tab，向后兼容
4. 更新到工作流 3.4 节


v1.4
【问题】：HTML原型视觉效果一般，想接入 Pencil 设计工具画更精美的高保真原型

【思路】：
1. Pencil 作为可选增强，不作为必须依赖——没装Pencil也能跑通整个工作流
2. HTML原型完成后主动询问用户是否需要Pencil增强，让用户选择
3. 自动检测Pencil安装状态，未安装给安装引导，已安装检测MCP连接
4. 建立统一设计变量体系（$primary、$accent等），Pencil和HTML两端保持一致
5. 新增 HTML↔Pencil 视觉一致性校准步骤，避免PRD中左右截图不一致
6. 新增 MCP 连接故障排查方案（含WebSocket直连降级）
7. 更新到工作流 3.5 节
